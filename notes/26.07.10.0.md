# Frontier 26.07.10.0

### Improved
- An app that declares a very dark accent colour now stays legible in the dock — its selected and running states are lifted to a readable tint instead of washing out against the near-black rail.
- Session status now resyncs on reconnect (e.g. after a host restart), not only on refocus.
- Idle/backgrounded apps no longer refetch the worker list on every worker heartbeat.
- none (developer-facing: the frontend and worker now typecheck in CI).
- The Workers list now shows a machine's hostname instead of "Unnamed worker" when no title is set.
- none (developer-facing: the host now typechecks in CI).
- A worker with no name set now shows its machine's hostname in the pickers and the add-workspace dialog, instead of its internal id.
- Bus channels renamed machines→workers (workers.list, worker.connected, worker_actions.*).
- The Host → Connections About tile now shows host uptime.
- Session status is now push-driven; idle apps stop polling for it.
- Background windows no longer poll the host; views resync on focus.
- Deleting a workspace now also removes its unused warm/pooled working copies from the worker, instead of leaving them on disk forever.
- A chat tab that couldn't reserve a workspace slot now clearly shows it holds no slot (instead of looking active); reopening it retries.
- The add-workspace dialog now shows a worker's name or address instead of its internal id.
- Deleting a workspace now releases any work areas still open on it, so it no longer leaves orphaned "Unknown workspace" slots behind.
- Fixed a rare workspace slot leak where closing a chat tab just as it was opening (or right after a reload) could leave the slot reserved with no tab.
- The Host connections view no longer labels a worker or surface reached over an internal or on-box relay "public network"; it now matches the Links section and calls it "local network".
- The Editor rail no longer shows a raw workspace id when a work area outlives a deleted workspace; it reads "Unknown workspace" and the dangling work area stays visible so you can close it.
- The editor's left nav drops its redundant "Workspaces" title, and its empty stage matches the calm icon-watermark used across the other apps.
