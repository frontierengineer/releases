# Frontier 26.07.07.0

### New
- The Pair a device screen now offers a Copy pairing link button, so a display or dashboard panel that cannot scan a QR can be paired by pasting the link.
- Added a dashboard app showing the host's machines, sessions and workspaces at a glance, and a pair-once wall-panel surface that displays it full-screen on any spare screen.
- When adding a workspace, the directory field now suggests folders on the machine as you type, with keyboard completion.
- Creating a chat now lets you pick the model and reasoning effort for the chosen runtime, plus any other option the runtime offers.

### Improved
- Assistant tabs opened with a single click stay in preview mode until their header is double-clicked to keep them open.
- Panel side columns can be resized smaller, hold at their minimum width while dragging, and collapse only when dragged to the edge; a short drag reopens a collapsed column.
- Machine view tooltips and dialogs now use plainer, more consistent wording.
- Side panel rails can be resized smaller, hold at their minimum width while dragging, and collapse only when dragged to the edge; a short drag reopens a collapsed rail.
- The welcome screen now detects a Frontier host that is already installed or running on this computer and offers to connect to it instead of setting one up again.
- The Machines view now explains slots and reservations accurately and no longer shows a host name line above the machine list.

### Fixed
- The editor's workspace list now shows which machine each workspace is on, and no longer comes up empty after a restart.
- Host setup now uses an existing Python 3.10 or newer when the private Python runtime cannot be installed, instead of failing the install on offline or unsupported machines.
- Renaming a host now updates its name for everyone connected to it, and the host list shows each host's real name instead of "This computer".
- Setting up a host now installs everything voice and speech control need up front, and voice recovers automatically instead of failing silently after a restart.
- The Files, Editor, and Git views now work on any workspace without a running agent: browse a workspace's files, open an editor work area on demand, and view branches, stage, and commit in git.
- The status bar now adapts cleanly to narrow windows, and the voice unavailable indicator is red.
- Setting up a host over SSH or with the install script now completes and reports success correctly, and hosts set up over SSH appear in the recent hosts list.
- Help hints in the bottom bar now stay readable while any dialog is open, whatever their length.
- Double-clicking a preview tab's header now keeps it open as a permanent tab.
- Spaces now recover cleanly after a failed turn, and subspaces created by agents always get a runtime, start active, and begin working their first direction immediately.
- Dragging an app icon in the left rail shows the insertion line again.
- Help hints in the bottom bar now stay readable while a dialog is open, whatever their length.
- The editor's Settings entry no longer shows a doubled separator line at the bottom of the file list.
- The Claude sign-in indicator now respects a relocated Claude configuration directory.
- The guide now reconnects to its existing session after a reload instead of creating a duplicate session each time.
- Terminals in the Machines view now restore their screen and recent scrollback after reloading the app, instead of coming back empty.
- Re-running the host installer on Linux now restarts the service so the freshly installed version takes effect immediately.
